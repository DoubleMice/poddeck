# 微软的"强制排名"往事

<div class="text-sm leading-relaxed mt-2">

TBPN 主持人回顾了 Xbox 所处的更大背景——微软自身的文化演变。

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-400 text-sm">

### Stack Ranking 之殇

- 管理者被迫将员工按钟形曲线分类
- 即使全明星团队也要划出"最后 10%"
- Vanity Fair 2012 年报道引用员工评价：**"微软内部最具破坏力的制度"**
- 让员工与同事竞争，而不是与对手竞争

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-400 text-sm">

### 从摧毁到重生

- 1970s-80s：Bill Gates 亲审工作，"令人激动但也不安"
- 1990s-2000s：强制排名制度化，Balmer 时代股价停滞
- 2013：Nadella 上台前废除强制排名
- 2026：Ashish Sharma 试图重建"激动且不安"的文化

</div>

</div>

<div class="mt-3 text-xs opacity-50">

作者概括：2014 年 Nadella 上任后宣布了 18,000 人裁员（多与 Nokia 手机业务整合有关）。微软的真正第一次大规模裁员是 2009 年金融危机时，此前 30 多年从未大规模裁过员。

# 🏎️ Ferrari：12 缸、三踏板、手动挡回来了

<div class="mt-4">

自 2012 年以来，**Maranello 首次推出带档杆的手动挡法拉利**——12 Cilindri Manuale。

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

### 关键参数

- V12 发动机，**9,500 rpm 红线**
- 6 速手动模式（原本是 8 速 DCT）
- 离合器踏板 + 档杆 + 琥珀色背光
- **真的会被你开熄火**
- 系统会阻止你"money shift"换错档
- 售价～$500,000 起，限量 1,499 台

</div>

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm">

### 争议焦点

这是"**真正的**"手动挡吗？  
本质上是一个 DCT 变速箱加上电控手动接口——**manual by wire**。  
Ferrari 表示：**客户愿意用性能换老式驾驶体验的乐趣**。  
极速从 211mph 降到 196mph——但这正是卖点。

</div>

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 text-sm">

- **"砍掉手工项目"**——不做人类太空飞行，不做第三方发射

</div>

- 失败一次损失自己的东西，而不是 NASA 的
- 所以可以"尽早失败、频繁尝试"
- AI 计算本身的经济价值密度足够高，一次性火箭也可能有经济可行性

# 数据中心投资：VC 和房地产的混合体

<div class="text-sm leading-relaxed mt-4">

Daniel English 描述了数据中心投资的独特性质——不是传统的 PE 房地产，也不完全是 VC。

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

### 规模

- 单笔交易典型规模：**$6 亿美元**（这是"小"的）
- 低端 $5 亿，高端可达 **$100 亿**
- 租户（超算中心）投入的芯片和基础设施也有数亿美元
- 双方都有巨大的资本押注——高度合作伙伴关系

</div>

<div>

### 风险特征

- **资本方是投机性的**：不是先签租户再买地，而是先买地、先建设、再找租户
- 租户也很紧张——所有人都抢同样的空间，超算中心、Neo Cloud、游戏公司竞标同一栋楼
- 有一点像 VC：需要有专业知识才能做风险调整分析

</div>

</div>

# 🖥️ Michele Catasta：后 Prompting 时代已经到来

<div class="text-sm leading-relaxed mt-2">

Replit 总裁兼 AI 负责人 Michele Catasta，是 vibe coding 领域的先行者——Replit Agent 项目在"vibe coding"这个词被发明之前就已经开始了。

</div>

<div class="mt-4 p-4 rounded bg-blue-50 border-l-4 border-blue-500 text-lg italic">

"我们不再花太多精力在 prompt 上了。即使你去听 Anthropic 的研究者和工程师讲解如何使用 Fable 5，核心洞察是：**停止给它指令，停止给它规则。告诉模型你想达成什么目标，然后相信这个过程。**"<br/>
<span class="text-sm not-italic opacity-70">— Michele Catasta</span>

</div>

<div class="mt-4 text-sm">

Michele 将这种现象称为"**后 prompting 时代**"——从给模型精确的指令，到只告诉它你想要的结果。

</div>

# 从 Vibe Coding 到 Vibe Building

<div class="grid grid-cols-2 gap-4 mt-4 text-sm leading-relaxed">

<div>

### 2024-2025：Vibe Coding
- 用自然语言生成代码
- 主要受众：独立开发者和创客
- 产出：一次性软件和内部工具
- Replit 率先发布 Agent 产品（2024 年 9 月）
- "vibe coding" 这个词到 2025 年 2 月才被造出来

</div>

<div>

### 2026-2027：Vibe Building
- Agent 不再是帮你写代码，而是**完成任务**
- 你设定目标（如"提高转化率 5%"），Agent 自己决定怎么做
- 可能是改网站文案、换图片、调整定价页
- "loops"——持续测量直到达成目标
- 一种全新的计算范式正在浮现

</div>

</div>

<div class="mt-4 text-sm opacity-60">

**作者概括**：Michele 将这种转变类比为从"教电脑怎么做"到"告诉电脑做什么"——这正是 AI Agent 和传统软件工程之间最本质的区别。

</div>

# Agent 的三个场景：不同成本结构，不同选择

<div class="grid grid-cols-3 gap-3 mt-4 text-center text-sm leading-relaxed">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">

### 广告位：一次性任务
"帮我抓房地产数据，整理到表格，做成信息图"  
→ 每次都用最前沿模型，按 token 付费  
→ 像雇了一个超级执行助理

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">

### 持续运行的 Agent
"每天帮我总结业界动态"  
→ 需要持续调用 LLM  
→ 成本较高但仍可接受

</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500">

### 确定性软件
"帮我做一个库存管理系统"  
→ 建成后不再需要 AI 推理  
→ 成本极低，可以给全公司用  
→ 目前 Replit 上最多的用例

</div>

</div>

<div class="mt-4 p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm">

**Michele 的忠告**：如果用 Agent 替代确定性软件的部分功能，你正在用 token 成本换取灵活性。一个被几万人使用的内部工具，如果每次都走前沿模型推理，账单会很可怕——所以在可预见的未来，前两种模式会并存。

</div>

# 🤖 AI 好奇心 vs 算法信息流

<div class="text-sm leading-relaxed mt-4">

Perplexity CEO Arvind Srinivas 在 Joe Rogan 节目中提出的二分法，引发了 TBPN 的讨论：

</div>

<div class="mt-4 p-4 rounded bg-blue-50 border-l-4 border-blue-500 text-lg italic">

"有两条通往好奇心的路，一条会杀死它，一条会放大它。杀死好奇心的，是**算法信息流**——每天被喂给你的脑腐内容，无休止的下滑。而能放大好奇心的，是 **AI**——现在你可以问任何你想问的问题。"<br/>
<span class="text-sm not-italic opacity-70">— Arvind Srinivas, Perplexity CEO, on Joe Rogan</span>

</div>

<div class="grid grid-cols-2 gap-4 mt-4 text-sm">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-400">

### 算法信息流 = Push
- 平台推什么你看什么
- 优化目标是注意力和留存时间
- 用户是**被动的**
- 看完了后悔——"这不是我真正想要的"

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">

### AI = Pull
- 你想知道什么就问什么
- 你主动决定探索方向
- 用户是**主动的**
- 对好奇的人来说极其满足

</div>

</div>

# "我的灵感都来自信息流"

<div class="text-sm leading-relaxed mt-4">

TBPN 的 Jordy 对 Arvind 的二分法提出了反对意见：

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm">

### Jordy 的反驳
"有时候好的想法确实在什么都不做的时候冒出来——对我来说是桑拿，有些人是淋浴。但我**大量的灵感来自信息流**。昨天我在 X 上看到一条链接，马上发给 John 说我们应该做这件事。"

"在互联网普及之前，艺术家找灵感也是去买书、看电影、消费媒体——信息流就是现代版的那个。"

</div>

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm">

### 折中观点
Jordy 的框架：**信息流负责灵感，AI 负责执行**。

一旦有了灵感，就用 AI 去深入探索——"这种事以前是怎么做的？这个方向的先例是什么？"
</div>

</div>

<div class="mt-4 text-sm opacity-60">

**TBPN 团队共识**：问题不在于"信息流"本身，而在于**算法的优化目标**是注意力而不是好奇心。StumbleUpon 那种随机发现 + 可控焦点的模式，可能是更好的平衡。

</div>
