# 为什么 Anthropic 能跑出来？

<div class="text-sm leading-relaxed">

**组织形态决定阶段优势**

代雨森在硅谷交流后的核心观察：AI 探索期适合自下而上的发散型组织（OpenAI），而高速竞技期适合自上而下、价值观对齐的聚焦型组织（Anthropic）。

- Anthropic 有价值观面试、Dario 每两周全员 memo
- 方向高度对齐，很像中国战斗力强的公司
- "管杀也管埋"——几十天迭代几十个功能
- 产品迭代极快，每周都在升级

# "管杀也管埋" vs "管杀不管埋"

<div class="grid grid-cols-2 gap-4 mt-4 text-sm">

<div class="p-4 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**Anthropic：管杀也管埋**
- Claude Code 几十天迭代几十个功能
- Desktop 端经常打开就发现又升级了
- 选了 coding 方向后持续深耕
- 有明确 bet，从上往下推进

<div class="mt-2 text-xs opacity-60">
"它是一个管杀也管埋的组织"

</div>

**OpenAI：管杀不管埋**

# 智力阈值：烧到 100°C 才有蒸汽机

<div class="mt-3 p-4 rounded bg-blue-50 border-l-4 border-blue-500 italic">
"把水烧开了才有蒸汽机，那如果烧到 99 度，其实也不会有蒸汽机。但是这个道理多少度是烧开，这个提前预计我觉得是比较难的。"
<div class="text-xs opacity-60 mt-1 not-italic">— 代雨森</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 leading-relaxed">
**为什么提前预测阈值很难？**
- Claude 4.5 不叫 Claude 5——说明 Anthropic 自己也没有那么大预期
- 没有新架构，没有新训练方式
- 但 4.5→4.6 带来了 coding 体验的本质变化
- 事后看节点很明显，事前没人能预测
</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 leading-relaxed">
**为什么偏偏是 coding 突破？**
- 数据——高质量用户 coding 数据让模型编程能力跃升
- agentic RL——让原本跑不起来的 agentic loop 跑起来了
- AI 开始能完成更长时间、更高价值的编程工作
- Coding 不是垂直领域，是 horizontal——能加强医疗、研究、办公等一切
</div>

# 智力阈值：烧开水类比

<div class="text-sm leading-relaxed">

**智力必须突破阈值才有价值**

把 AI 的智力从猫提高到智商 80 是重大成就，但没有商业价值——就像 99°C 的水不会产生蒸汽机。

- 阈值突破前：AI 只能写诗、翻译——"套壳"
- 阈值突破后：agentic coding 可用——Harness 变得重要
- 突破是突变，不是渐变
- 多少度算"烧开"无法提前预测

</div>

# Return 链条：产出 ≠ 结果

<div class="text-sm leading-relaxed">

**投入→产出 这段是通的**
Token 确实能烧出软件来

**产出→结果 这段还没通**
软件做出来了，但变成利润了吗？

- 程序员突然多了 10 倍，公司的收入会大涨吗？
- 很多时候不是缺程序员——是不知道该做什么
- 硅谷公司开始互相"卷"进彼此的领地
- Loveable 做 PPT，Gamma 做网站
- 复制已有功能越来越快，但创新还是很难

# Token 烧了 ≠ 赚了钱

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500 leading-relaxed">
**降本是裁员——但只能裁一次**
- 程序员 80-90% 的工作是编程，但 10-20% 是沟通、协调、信任
- 大公司随时可以裁 15% 冗余——AI 来不来都能裁
- 进一步裁没那么简单——就像自动驾驶解决了 80% 高速场景也不能去掉人
- 裁员只能裁一次，不能持续、增量地省出成本

</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 leading-relaxed">
**增效做新产品——是个慢过程**
- 移动互联网停滞——不是缺程序员，是缺好产品经理发现新需求
- 烧掉的 Token 是现在就烧掉的——赚回来可能要两年
- 水槽比喻：上面吃螃蟹的人远多于下面觉得螃蟹不好吃离开的人
- 漏水不堵住，ARR 涨得再快也是不可持续的

</div>

</div>

<div class="mt-3 p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
**时间错配**：Hyperscaler 已经在举债建设数据中心——用资金杠杆后，对回报问题解答的时间要求反而在变短。Too big to ignore。
</div>

# Harness：从"套壳"到"编排工程"

<div class="mt-3 p-4 rounded bg-blue-50 border-l-4 border-blue-500 italic">
"去年大家都在说这是套壳不重要，现在大家说啊，原来叫 harness 变得很重要。我就觉得大家对这个事情的判断，对它的重要性的判断，其实都是在不断的变化的。"
<div class="text-xs opacity-60 mt-1 not-italic">— 代雨森</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**Harness 是 F1 车队的维修站**
模型是车手，开得很好——但需要有人保养、换胎、控制赛道。Harness 提供：Context（实时信息 + 组织专有数据）+ Tools（可使用的工具）+ Agent Loop（任务循环机制）
</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 leading-relaxed">
**Harness 创新的例子**
- Claude Code CLI：为 AI 而非人类设计的界面
- OpenClaw 心跳机制：每 30 分钟检查未完成任务
- OpenClaw Memory/MDP：所有对话在一个大聊天里，AI "记得你"
- 插到微信/WhatsApp/Discord——去用户熟悉的地方
</div>

# Harness = OS · Model = CPU

<div class="text-sm leading-relaxed">

**类比 Windows 的诞生**
在 DOS/Windows 出现前，写应用要自己处理跟 CPU 的通讯。Windows 出现后，开发者只需处理 API。

- Model (CPU) 层：Claude / GPT / Kimi——可插拔
- Harness (OS) 层：Claude Code / Codex / OpenClaw——处理 agent loop、context、memory
- 应用层：Skills、基于 Harness 的 App

**三年前 AI 应用开发者要自己写 agent loop、memory、guardrail。现在 Harness 帮你处理，你只管做 skill。**

</div>

# 壳变得更值钱了

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 leading-relaxed">
**数据飞轮：壳反哺模型**
- 好的 Harness 收集高质量用户交互数据 → 训练更好的模型
- Cursor 用其 Composer 收集数据，基于 Kimi pre-training + 自有 feedback → 做出好模型
- 用户不是为了模型忠诚，而是为了 Harness 里的记忆忠诚——不断"救活小龙虾"

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**用户粘性在 Harness 侧**
- 在 Claude Code 里配了 skill、写了 CLAUDE.md → 换 Codex 得重新弄
- 记忆存在 Harness 里——"换模型可以，换 Harness 不行"
- Codex 靠价格战（便宜 50%）+ GPT 5.5 追赶
- 品牌锁定：用惯了就不想换

</div>

</div>

<div class="mt-3 p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
**模型级产品 vs 壳**：这两个观点可以同时成立。好的产品需要好的模型提供核心能力，但好的 Harness 也能通过数据闭环反哺模型——Cursor 训出了 Composer，没有壳就没有这个模型。
</div>

# Agent 进入人类世界的三步走

<div class="text-sm opacity-70 mt-3">代雨森的框架——AI 像一个外星人来到人类世界：</div>

<div class="mt-4 space-y-2">

<div class="p-4 rounded bg-green-50 border-l-4 border-green-500">
<div class="text-sm font-bold">Step 1：让人类拥有更多、更好的 Agent</div>
<div class="text-xs opacity-70 mt-1">Claude Code、Codex、OpenClaw、Manus——降低安装和配置门槛。让更多人低成本拥有 Agent。这是当前的主战场。</div>
</div>

<div class="p-4 rounded bg-yellow-50 border-l-4 border-yellow-500">
<div class="text-sm font-bold">Step 2：让 Agent 更好地适应人类的数字世界</div>
<div class="text-xs opacity-70 mt-1">给 Agent 发卡（Stripe/Coinbase）、办认证（Cloudflare 不再 block AI）、用 Computer Use 和 Sandbox 操作人类已有软件。像一个外国人来到中国——需要 SIM 卡、微信、支付宝。</div>
</div>

<div class="p-4 rounded bg-blue-50 border-l-4 border-blue-500">
<div class="text-sm font-bold">Step 3：建设 Agent 专属的原生数字世界</div>
<div class="text-xs opacity-70 mt-1">Agent-native 的支付（高频、小额、多对多）、Agent 用 CLI/API 而非 GUI、Agent 间通信不需要 Excel。这是长期最大的创业机会。</div>
</div>

</div>

# Agent 之间的网络效应

<div class="mt-3 p-4 rounded bg-blue-50 border-l-4 border-blue-500 italic">
"六个月前我是没有理由雇佣你的 agent 的，因为你的 agent 跑的也是 open source，我的也是 open source。但现在，你的 agent 里积累了你所有的访谈经验、问题准备、思考——它做出来的东西就是跟别人不一样。"
<div class="text-xs opacity-60 mt-1 not-italic">— 作者概括代雨森的核心设想</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**差异化来自专有 Context**
- 你在 Harness 上沉淀的 skills、对话历史、偏好——让同一个任务给你做 vs 给我做得到不同结果
- 沈南鹏的 agent 看 BP 比戴雨森的 agent 更值钱
- Skill 一做成公开版就失去稀缺性 → 不如让 agent 接单

</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 leading-relaxed">
**一种 Agent-native 的商业模式**
- 我的 agent 雇佣你的 agent——1000 块 token 费 + 9000 块专有知识溢价
- 有点像电商 marketplace
- 已有创业公司在尝试 agent-to-agent marketplace
- 长期价值：体现 agent 的差异化

</div>

</div>

# 移动互联网的金科玉律正在失效

<div class="mt-3 p-4 rounded bg-yellow-50 border-l-4 border-yellow-500 italic">
"移动互联网的商业模式可以叫做 attention is all you need——DAU × 时长 × 变现效率 = 收入。但 Agent 的时代，用户自己看着屏幕的时长已经不是最关键的指标了。"
<div class="text-xs opacity-60 mt-1 not-italic">— 作者概括代雨森的观点</div>
</div>

<div class="grid grid-cols-3 gap-3 mt-4 text-sm text-center">

<div class="p-3 rounded bg-red-50 border border-red-200">
<div class="font-bold text-red-600 mb-1">指标变了</div>
<div class="leading-relaxed text-xs">从 DAU + 时长 → 一个 Agent 能持续完成有价值任务的时长</div>
</div>

<div class="p-3 rounded bg-blue-50 border border-blue-200">
<div class="font-bold text-blue-600 mb-1">护城河变了</div>
<div class="leading-relaxed text-xs">封闭 App 反成软肋——Agent 访问不了微信但能访问飞书，用户会迁移</div>
</div>

<div class="p-3 rounded bg-green-50 border border-green-200">
<div class="font-bold text-green-600 mb-1">获客方式变了</div>
<div class="leading-relaxed text-xs">领先 AI 应用不靠投放和补贴，靠"魔法体验"传播——ChatGPT 到 Claude Code 都如此</div>
</div>

</div>

<div class="mt-3 p-3 rounded bg-orange-50 border-l-4 border-orange-500 text-sm leading-relaxed">
**设计理念变了**：当用户不只是人，还有 Agent——你的 CL 命令行或 API 要做得更好，而不是 GUI 做得更花哨。
</div>

# DAU 高 ≠ 产品好

<div class="mt-3 p-4 rounded bg-red-50 border-l-4 border-red-500 italic">
"Claude Code 的 DAU 肯定远低于 ChatGPT，但它的收入量级可能要高很多。"
<div class="text-xs opacity-60 mt-1 not-italic">— 作者概括代雨森</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**高价值任务产品**
- 用户不需要天天上来盯着
- 睡觉前把任务布置下去，醒来就做好了
- 关键在于"解放注意力"——attention is NOT all you need
- 长程运行能力比日活更重要

</div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 leading-relaxed">
**如果 DAU 是北极星指标**
- 会出现荒谬的场景：为了优化 DAU，让用户不得不多上来看一看
- 但最好的体验是：布置完任务 AI 自己去跑，你都不用看
- 用户少访问 → DAU 低 → 但价值更高
- 指标和体验的矛盾

</div>

</div>

# 下一个字节不像字节

<div class="mt-3 p-4 rounded bg-red-50 border-l-4 border-red-500 italic">
"在字节的游戏规则里打败字节是非常难的。把上一个时代的成功范式搬到 AI 时代，你其实反而就是在跟上个时代的冠军去竞争。"
<div class="text-xs opacity-60 mt-1 not-italic">— 作者概括代雨森</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500 leading-relaxed">
**AI 信息流——新瓶装旧酒**
- 一打开是信息流，刷 AI 生成的内容
- 用推荐算法 + 买量 + 广告变现——字节最擅长的全套体系
- AI 小游戏、AI 短剧——用户要的是好玩的游戏，不是"AI 的"游戏
- Kill time 场景的 AI 应用：一上来就要比抖音/红果/小红书更好看

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**OpenClaw 的范式——完全不同**
- 没有自己的 App——活在微信、Telegram、Discord 里
- 去用户熟悉的地方，而不是把用户圈进来
- Agent 不需要信息流——它需要能访问你的数据
- 生产力场景：从手打 Excel 到 AI 帮你做，直接 10x 提升

</div>

</div>

# 新技术解决老问题 vs 渗透率后的全新疆界

<div class="text-sm opacity-70 mt-3">代雨森的历史规律总结——每次技术革命分两个阶段：</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 leading-relaxed">
<div class="font-bold mb-1">阶段一：新技术解决老问题</div>
- 互联网：门户网站（报纸上网）、email（邮件上网）、自营电商（商业上网）
- AI：AI 写文章、做翻译、画图、扣图——替代人类已有的工作
- 有价值，但不是最大的机会
- **AI 用人类的软件**——Computer Use 操作现有 GUI

</div>

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 leading-relaxed">
<div class="font-bold mb-1">阶段二：渗透率带来的全新机会</div>
- 互联网：Social Network、搜索引擎、平台电商（都是创业公司做的）
- 移动互联网：短视频/直播、推荐引擎、手游、O2O、拼多多
- AI：Agent 间协作、Agent-native 支付、组织变革、广告模式消亡……
- **不再需要人类的软件**——Agent 直接通信

</div>

</div>

<div class="mt-3 p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">
**现在处于**：第一阶段很多人在做，第二阶段开始出现雏形——代雨森投资的 Slock（人与 Agent 协作平台）就是早期探索。
</div>

# 蒸汽机 → 电动机：组织变革

<div class="text-sm leading-relaxed">

**技术不会自动带来生产力提升**

蒸汽机工厂围绕一根主轴建造，狭长形，皮带传动——无法做流水线。电动机发明后，不需要主轴了，厂房可以更大更扁平，这才诞生了流水线。

**AI 时代的对应**：
- 传统开发 = 瀑布式分工（前端/后端/测试/UI/运营）
- AI-native = 几个人负责一个产品模块，全栈 + Agent 协作，职能边界模糊
- Slack 自己就是用 Slack 写的，Cursor 是用 Cursor 写的

</div>

# 2026 年创业：Horizontal > Vertical

<div class="mt-3 p-4 rounded bg-green-50 border-l-4 border-green-500 italic">
"硅谷每个人都说我要做 vertical。但我认为在技术发展早期变化太快了，如果你做得太垂直的话，你是容易把自己给困进去的。"
<div class="text-xs opacity-60 mt-1 not-italic">— 作者概括代雨森</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 leading-relaxed">
**为什么现在要做 Horizontal**
- 技术变化快——做通用平台能承载不同场景
- Manus 一开始不知道做 PPT/网站——做了通用 agent 后自然衍生出多个场景
- Perplexity、Manus、GenSpark、OpenClaw——跑出来的 AI 应用大部分是 horizontal
- 垂直 SaaS 是 SaaS 后期的事——通用机会被做完了才做垂直

</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 leading-relaxed">
**给创业者的两个 Takeaway**
1. 做真正大幅度的创新——变化快、机会多，最怕为了稳妥去跟风
2. 敢于做 horizontal——当技术变化很快时，horizontal 能让很多新浪潮对你有利
3. 缺钱可以融资，创新失败可以再来——但跟着别人跑、做不创新的事，时间就浪费了

</div>

</div>

# AI Native 创业者：出题家 vs 做题家

<div class="grid grid-cols-2 gap-4 mt-4 text-sm">

<div class="p-4 rounded bg-red-50 border-l-4 border-red-500 leading-relaxed">
<div class="font-bold text-red-600 mb-2">不投的：跟风的"做题家"</div>
- 什么赛道火了就做什么——可穿戴录音赛道、各种 Menlo
- 做一个只是"better"的产品，没有原创判断
- "我们希望投的是出题家——提出了一个不一样的看法，做不一样的东西，哪怕这个东西看上去很奇葩"
- 代雨森过去一年 pass 最多的就是跟风项目

</div>

<div class="p-4 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
<div class="font-bold text-green-600 mb-2">要投的：能产生原创判断的"出题家"</div>
- 在别人还没有谈这个概念之前就去做的——像杨植麟 2021 年就要训大模型
- 对技术有深刻理解 + 企业家精神
- 能产生原创判断——不是听播客听来的
- 适应得早、对不同技术路线的细微差别有深刻认知
- 真格投了两家世界模型公司——不是因为风口，而是跟了创始人两三年

</div>

</div>

<div class="mt-3 p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
**底层能力不变**：学习力、创造力、领导力、意志力——从徐小平做新东方到现在，这些不变。变的是对技术的理解——二十年前是互联网，十年前是移动互联网，现在是 AI 和机器人。
</div>

# 思考需要刻意练习：思想的健身房

<div class="mt-3 p-4 rounded bg-purple-50 border-l-4 border-purple-500 italic">
"你可以外包思考，但是不能外包理解。"——安德斯·卡普西
<div class="text-xs opacity-60 mt-1 not-italic">— 代雨森引述</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 leading-relaxed">
**思考不能外包**
- 外包体力 → 身体变弱 → 有了健身房
- 外包记忆 → 记性变差 → 搜索代替
- 外包思考 → 思考能力退化 → 需要"思想的健身房"
- 做播客逼自己思考——观点里有什么论据？从哪里来？不糊弄过去

</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 leading-relaxed">
**创新也需要刻意练习**
- "林总"每个月烧一万美金 Token——不是在浪费，是在刻意练习用 AI 创新
- 扫描域名、注册、给每个域名做东西——像 omakase，今天有什么鱼就做什么菜
- 从简单的 web app 开始：同步会议、同步日程、同步订阅
- "很多东西确实好像不是直接能产生价值，但在这个过程中我知道用 Agent 协作真正的难点在哪儿"

</div>

</div>

# OOD：人类最后的堡垒？

<div class="mt-3 p-4 rounded bg-red-50 border-l-4 border-red-500 italic">
"AI 能很好解决 within distribution 的事情——写代码尤其是日常写代码，因为是分布内的。但 AI 讲不出一个原创的笑话，它只能把人类已经讲过的笑话重新表达。"
<div class="text-xs opacity-60 mt-1 not-italic">— 作者概括代雨森</div>
</div>

<div class="grid grid-cols-2 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 leading-relaxed">
**AGI 在缩水**
- 最初：毁灭人类 / singularity
- 后来：解决黎曼猜想、创造新东西
- 现在：替代普通程序员
- "AI 毁灭旧价值的速度，远高于创造新价值的速度"
</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
**什么是 AI 还做不到的**
- 创造 out-of-distribution 的内容——伽罗华 20 多岁搞群论
- Taste 也不是护城河——AI 见过的数据远超人类
- **Agency**：知道要做什么、想改变什么、能问出好问题
- 如果你的能力全在正态分布中心——很容易被替代

</div>

</div>

# 硬件、机器人与世界模型

<div class="text-sm opacity-70 mt-3">代雨森在硅谷看到的三大前沿方向 + 对中国市场的观察：</div>

<div class="grid grid-cols-3 gap-3 mt-4 text-sm">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 leading-relaxed">
<div class="font-bold mb-1">Coding 大跃进</div>
硅谷全民烧 Token——Meta 有 leaderboard 比谁烧得多。有兴奋（AGI 即将实现），有恐惧（失业），有 FOMO（想做 Anthropic 股东）
</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">
<div class="font-bold mb-1">世界模型</div>
把语言模型的范式平移到机器人：Pre-training + Fine-tuning + RL。通过大规模 ego-centric 视频数据训练，让机器人具备 zero-shot 泛化能力。但"世界模型"定义尚未收敛。
</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 leading-relaxed">
<div class="font-bold mb-1">Auto Research</div>
AI 自我迭代、AI 提升 AI——科学研究 + AI 研究。Kim 的 Cursor 在做自我改进的 AI。让训练速度变得更快。
</div>

</div>

<div class="mt-3 p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
**代雨森的谨慎区域**：消费电子/AI 可穿戴硬件——"很多会重蹈新消费覆辙"，把传统品类加个 chatbot 不等于新品类。好的载体可能还是手机和云端。
</div>
