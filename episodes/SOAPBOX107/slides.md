# 确定性 Controls + AI 上下文

<div class="text-sm leading-relaxed mt-2">

Tony 的核心论点：**AI 不应该取代确定性的安全检测工具**，而应该在确定性结果之上提供上下文和关联。

**确定性层的优势**：
- 云配置的对错是 API 返回的布尔判断，不是 AI 的猜测
- "if S3 bucket is open, raise a flag" — 这是确定的
- 社区持续贡献新的 controls，覆盖范围不断扩大

**AI 层的价值**：
- 关联多个 findings 形成攻击路径
- 用 Graph DB 连接所有结果，呈现全景图
- 理解上下文："open because it has to be, no worries; open with PII, big red flag"

</div>

# Build vs. Buy：AI 时代的新变数

<div class="grid grid-cols-2 gap-4 mt-4 text-sm">

<div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 leading-relaxed">

**Build 的诱惑**

"Now you can build in a week what you could build like ten years ago in five months."

AI 让 build 的成本急剧下降。企业开始问："为什么我要为 SaaS 付费，如果可以在内部用 AI 搭一个？"

</div>

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500 leading-relaxed mt-3">

**Tony 的回应**

"This is the risk of building a successful open source project, but not from today because of the AI. Also, ten years ago was the same issue."

开源项目一直面临被 fork/包装的风险 — AI 只是让这个过程更快了。

</div>

</div>

<div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 leading-relaxed">

**Buy 的价值**

"I think most people they can't be bothered doing that. They're probably going to just spend the money."

拥有成功的项目和活跃的社区本身就是护城河 — 重复造轮子没有意义，尤其是当已有工具和社区已经存在时。

"What you're really paying for is the expertise, and that's not changing."

SaaS 中 "as a service" 的核心是服务 — 持续的更新、维护、和专业知识，这不会因为 AI 而消失。

# The Frankenstein Problem

<div class="text-sm leading-relaxed mt-2">

Tony 认为 vibe-code 最大的风险不是"能不能造出来"，而是"能不能维护下去"。

"But we will see what's going on in a year from now, in two years from now, when those very well done, bite coded applications to solve a problem, are becoming a monster, are becoming a Frankenstein, and the two guys that were building that they decide to leave the company."

**这是历史上演的规律**：20 年前开源让软件构建更快了，但"I have a three terabytes MySQL database and call somebody to optimize those queries" — 拥有和运行是两个概念。

AI 让 build 变快，但 maintenance 不会变简单。

# 为什么不能用 LLM 直接管云安全？

<div class="mt-4 text-sm leading-relaxed">

<div class="p-4 rounded bg-red-50 border-l-4 border-red-500 leading-relaxed">

**Patrick 的警告**: "You can go ask a model, hey, you know, here's an API key, go secure my cloud infrastructure. And I think we know that's going to end really badly."

</div>

<div class="mt-4 p-4 rounded bg-blue-50 border-l-4 border-blue-500 leading-relaxed">

如果你试图让 LLM 直接管理云安全，你实际上是在给它一个 spec — 这个 spec 本身就是软件的第一版需求文档。

<div class="text-sm leading-relaxed mt-2">

**Prowler 的 MCP 集成**：
- "you can configure private MCP, which means that you make your AI-driven IDE cloud security expert"
- 从 Claude Code、Windsurf、Copilot 等 AI IDE 中直接连接 Prowler
- Agent 可以创建实时 dashboard、运行修复、自动发起 pull request

**Patrick 的类比**：
"You hear about Claude being used to do off sec stuff, but like, what are you going to give it netcat and curl and tell it to go do a pen test? Like, it's not going to work. It needs to use tools."

<div class="mt-3 text-sm leading-relaxed">

"That is why we call Prowler the Agentic Cloud Defender, because it's not only the human that can go through APIs and dashboards and beautiful charts, all that stuff, to see what's going on, but also an agent to say, to see, okay, this is the data, this is the correlation, give me what is next, right? # Three Sources of Truth

<div class="text-sm leading-relaxed mt-2">

Tony 把 Agentic Cloud Security 的确定性基础概括为三个来源：

**1. Detection Registry** — "what you understand for cloud security, which is the registry of artifacts to know what to detect"

**2. Remediation Guidance** — "if it's right or wrong, what to do" — 修复方案需要被明确告知，可以加 "a pinch of AI"

**3. Compliance Mapping** — "if you need to be compliant with something, that needs to be mapped" — AI 做 mapping 可能出错，需要有确定性的映射

**这三者的关系**：Agent 知道了什么是对错、知道怎么修复、知道影响哪些合规框架，就能在护栏内自主运行。

# 软件是商品，维护不是

<div class="mt-4 text-sm leading-relaxed">

<div class="p-4 rounded bg-yellow-50 border-l-4 border-yellow-500 leading-relaxed">

**Tony 的关键判断**: "Software is a commodity; software can be created by anybody. But software is not only creation; software is about maintenance, evolving, knowing why you are building."

"do you think a guy, of course, in a garage, can create a CrowdStrike, the new CrowdStrike? They can probably do effective XDR, right? You and I in a weekend can do some sort of XDR, but this is not about an XDR. Not an XDR I'd want to run."

</div>

<div class="mt-4 grid grid-cols-2 gap-3 text-sm">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">
<strong>创建 vs 维护</strong><br/>
<span class="opacity-70">AI 让任何人可以在周末写一个 XDR — 但写一个你愿意用的 XDR 是另一回事</span>
</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">
<strong>Prowler 的护城河</strong><br/>
<span class="opacity-70">庞大的社区、持续更新的 controls、跨云和 SaaS 的覆盖、确定性的 registry — 这些不是 vibe-code 能复制的</span>
