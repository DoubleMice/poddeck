# AI 存储器的黄金时代

<div class="mt-4">

SK Hynix 的 IPO 背后是 AI 对**高带宽存储器 (HBM)** 的井喷需求。HBM 是 AI 加速器的关键配套组件——没有高速内存，再强的 GPU 也跑不起来。

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">

**三大存储器巨头格局：**
- **SK Hynix** — HBM 市场领导者，独占 NVIDIA 高端 GPU 配套
- **Samsung** — 全球最大存储器厂商，加速追赶 HBM
- **Micron** — 美国唯一存储器巨头，HBM 份额第三

三家合计控制全球存储器市场，AI 需求让这一赛道的战略价值飙升。

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">

**为什么之前买不到 SK Hynix：**
- 韩国上市，需通过 KOSPI 交易
- 美国投资者面临跨境门槛
- ADR 上市后直接通过纳斯达克买入

这标志着美国资本市场对 AI 基础设施供应链的全面接纳——存储器不再只是"配角"，而是 AI 价值链的关键瓶颈之一。

# OpenAI 桌面应用：整合的阵痛

<div class="mt-4">

</div>

<div class="grid grid-cols-2 gap-3 mt-3">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500 text-sm leading-relaxed">

### 用户不满

- Codex 是 Electron 应用（Web 视图），不如原生 ChatGPT 应用流畅
- 原 ChatGPT 的快捷键、集成等习惯性功能被重新组织
- 没有原生聊天窗口——只有一个小型弹出窗
- 习惯特定工作流的用户被迫适应新范式

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">

### OpenAI 的回应

OpenAI 产品负责人发帖承认："过去 24 小时阅读反馈后，我们意识到没有完全做好几件事："

- 高端算力模式的用量限制不够清晰
- 桌面应用重组让聊天和项目更难找到
- Codex 用户担心产品被边缘化——"绝对没有这个意图，Codex 会保留"

</div>

</div>

<div class="mt-3 text-xs opacity-60">
**核心矛盾：** 用户想要一个"万能输入框"——像 Google 搜索一样统一。但产品层面，不同使用场景的集成方式仍在磨合。

# Phoebe Gates 的 FIA 风波

<div class="mt-3 text-sm leading-relaxed">

**Bloomberg 调查**揭露了 Bill Gates 女儿 Phoebe 联合创办的购物应用 FIA 涉嫌违规归因的操作方式：

</div>

<div class="grid grid-cols-2 gap-3 mt-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500">

### Bloomberg 的发现

- FIA 在用户结账时**无声打开后台标签页**，注入自己的推荐码
- 该推荐码覆盖了其他发布商的合法推荐链接
- 在 50 多个电商网站上测试，发现了相同的模式
- 独立研究员 Ben Edelman 和竞品 Capital One Shopping 复现了相同结论

</div>

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">

### 为什么比 Honey 更严重

- **Honey** 至少需要用户主动交互——即使是关闭弹窗也算一次点击
- **FIA** 完全不需要任何用户交互，只要安装了扩展就会触发
- 联盟营销的基本规则："只对真实的用户点击付费"
- Bloomberg 引用："模拟点击、虚构点击、想象点击都不行——只有真实的点击才算"

</div>

</div>

<div class="mt-3 text-xs opacity-60">
**定性：** 目前尚未被证明是刑事犯罪——可能只是一个未被发现的程序 Bug。但如果是故意的，则可能构成联盟营销欺诈甚至电信欺诈。
</div>

# 联盟营销的归因暗面

<div class="mt-2 text-sm leading-relaxed">

</div>

<div class="grid grid-cols-2 gap-3 mt-2 text-sm leading-relaxed">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">

### 🐴 正常流程

你在 **horsereviews.com（马评网）** 上研究骏马 → 点击链接跳转到 **clydesdalesdirect.com（骏马直销）** → 结账 → URL 中的 `utm=horsereviews.com` 记录归因 → 马评网获得佣金

<div class="text-xs opacity-60 mt-1">马评网做了教育工作，理应获得回报。</div>

</div>

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">

### 💉 Honey 模式（已被曝光）

你安装了 Honey → 结账时弹窗说"有优惠券" → 你点击（或关闭）→ Honey 悄悄替换归因标签为自己 → Honey 拿走佣金

<div class="text-xs opacity-60 mt-1">至少需要用户交互，争议点在于"最后点击归因"的公平性。</div>

</div>

</div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed mt-2">

### 💀 FIA 模式（Bloomberg 的指控）

你安装了 FIA → 你甚至不知道它在运行 → FIA 在你结账时自动打开后台标签页 → 自动注入自己的推荐码 → 静默覆盖其他发布商的合法归因 → **完全不需要你的任何操作**

</div>

# Honey vs FIA：归因劫持对比

**三种联盟营销归因模式。** 从公平到越界，关键差异在于用户是否参与了交互过程。

- Costco 股价自 2008 年以来上涨超过 **2000%**

<div class="mt-3 text-xs opacity-60">
**反直觉的结论：** 对一线员工慷慨可以成为卓越的长期商业策略。Costco 的创始人押注的就是这个逻辑——而事实证明，他们完全正确。

# Costco 战略飞轮
