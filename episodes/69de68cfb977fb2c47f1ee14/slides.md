# Anthropic：战略的胜利

<div class="mt-3 text-sm leading-relaxed space-y-2">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">

**从落后位置杀出**：2024 年夏天 Sonnet 3.5 给了 coding 正反馈之后，他们只 all in 了一件事——coding。今天做成了，而且巨大。

</div>

<div class="grid grid-cols-2 gap-3 mt-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">

**做了什么**
- All-in coding，放弃 C 端
- 完全放弃多模态
- 不跟风做 reasoning model
- 坚持 pre-train，不觉得撞墙
- Top-Down 战略，高度 aligned

</div>

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500 text-sm leading-relaxed">

**为什么别人做不到**
- 各 lab 最聪明的人都想做自己的 research bet
- 没人愿意做脏活累活（搞数据）
- 需要有 founder 亲自带队搞数据
- 需要"不做什么"的战略克制

</div>

</div>

<div class="p-2 rounded bg-yellow-50 border-l-4 border-yellow-500 text-xs mt-3">

"他们的几个 founder 可能一开始就意识到数据可能是所有问题的根本。" — 首席科学家 Jared Kaplan 亲自干数据活，这比较少见。

</div>

</div>

# Anthropic 的组织与文化

<div class="mt-3 text-sm leading-relaxed space-y-2">

<div class="grid grid-cols-2 gap-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">

**组织特点**
- 一帮人只做一件事，高度聚焦
- 不招 big name，招 underdog
- 文化面试极其严格（"AGI 实现后你会怎么做？"）
- 团队稳定，人才流失少
- 内部很透明，对外严格保密

</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 text-sm leading-relaxed">

**文化特点**
- 创始人特别 hands-on，懂技术
- 没有刻意神化某个人或某一环节
- 不做下一个范式，踏踏实实执行
- 更像"一个球队、一个工业化体系"，每个环节都做好
- 每个人都很相信 AGI，特别 AGI 原生

</div>

</div>

<div class="grid grid-cols-2 gap-3 mt-3">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">
<strong>对比 OpenAI</strong>：OpenAI 像 VC，喜欢造概念、神化某个环节（pre-train → post-train → o1 o3），一个人做一件新事，动荡摇摆大。
</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 text-sm leading-relaxed">
<strong>对比 Google</strong>：Google 更像机器或体系化运转，任意换掉几个人没啥影响，但自下而上的文化导致焦点分散。
</div>

# Anthropic 的产品哲学

<div class="mt-3 text-sm leading-relaxed space-y-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">

**不做 IDE，做终端** — Claude Code 是一条不同的路。Cursor 火了以后，很多人讨论模型公司要不要做个 IDE。Anthropic 没有，而是做了终端形式的 Claude Code，**因为终端更能承接住模型的红利**——模型是指数级增长的，产品也得指数级接住。

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">

**过去五十多个工作日发了七十多款产品和 features** — 这个速度在互联网时代根本做不到。做产品的人很多是工程师或 researcher，非常懂模型能力，把模型能力转化到产品体验的效率很高。

</div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500">

**定位高价格带** — 定价一直很高，不降价。因为定位的是塔尖用户（一两百万人），用最大模型、最好效果、最高定价、也有好 margin。算力可能是今年实现千亿美金 ARR 的最大 bottleneck。

</div>

</div>

# 做好 Coding 的两大支柱

<div class="text-sm leading-relaxed mt-4 space-y-2">

广密认为做好 coding 最重要的不是技术 know-how，而是两件事：

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 mt-3">

**1. 组织与文化** — 能否让几百个最聪明的人 all-in 搞 coding 一件事？因为有太多脏活苦活（搞数据），而聪明人都想做 0→1 的突破。

</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 mt-3">

**2. 数据壁垒** — Coding + Agentic 的数据比以前 chatbot 的数据复杂得多：既有任务、又有环境、又有评估。需要大量造数据，而且别人家的数据分布不够全。

</div>

# OpenAI：巨头的光芒与阴影

<div class="mt-3 text-sm leading-relaxed space-y-2">

<div class="grid grid-cols-2 gap-3">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">

**光芒**
- C 端遥遥领先：9 亿+ 周活，5000-6000 万付费用户
- 过去 lead 过两次范式级创新
- 下一次大概率也是 OpenAI lead
- 人才密度高、综合实力强
- Researcher 是一等公民，敢于冒险

</div>

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500 text-sm leading-relaxed">

**阴影**
- 在 coding 上有严重战略误判
- 忙着抢 C 端 DAU 而忽视 coding
- Sam VC 思维：摊大饼、什么都想要、容易 FOMO
- 文化 value 0→1 但不 value 1→100
- 没人做脏活累活，都想要突破性工作
- ChatGPT 成功但"没有灵魂"

</div>

</div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 mt-3 text-sm">

**作者概括**："OpenAI 最大的战略误判还是整个 pre-train 和 coding 这两个没做好。因为 Sam 不是技术出身，对技术路线或做哪些技术决策可能不如 Anthropic 这种技术出身的人更果断。"
</div>

# "今天胜利的秘籍，就是下个时代的毒药"

<div class="mt-4 text-sm leading-relaxed space-y-3">

<div class="p-4 rounded bg-blue-50 border-l-4 border-blue-500 italic text-lg">

"今天胜利的秘籍可能就是下个时代的毒药对吧？OpenAI 在 ChatGPT 过于成功，让他们专注 to C 而忽视了 coding。"

</div>

<div class="mt-4 text-sm leading-relaxed space-y-2">

**广密的判断**：

- OpenAI 的 coding 和 agentic 能力大概率能追上 Anthropic——最近战略大转向是好事
- 有 **50% 的概率** AGI 最终 winner 还是 OpenAI
- Anthropic 今天的成功更多是**执行的胜利、专注的胜利**，但未必能抵御 OpenAI 下一个范式级突破
- OpenAI 自下而上的文化下，一两个人可能搞出惊天动地的大事——**可能最终决定胜局的，就是一两个人**

</div>

<div class="p-2 rounded bg-yellow-50 border-l-4 border-yellow-500 text-xs mt-3">

"OpenAI 一直想做爱因斯坦，Anthropic 是把整个白领工作给 automate。你想跨过 coding 直接去做爱因斯坦，但这条路很难。"
</div>

# Google Gemini：被高估的 "王者归来"

<div class="mt-3 text-sm leading-relaxed space-y-2">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">

**Gemini 3.0 被高估了** — benchmark 刷得很高，但 C 端没有持续增长，用户也不太买单。除了让 Google 股价翻倍、证明不是 AI loser 了，**其他好像也没捞到太多好处。**

</div>

<div class="grid grid-cols-2 gap-3 mt-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">

**问题**
- Benchmark 过度优化
- 产品能力弱（Gemini 3 连 PC 桌面版都没有）
- 工程师/技术驱动，PM 弱势
- 严重误判 coding 的重要性（晚了 3-4 个月才意识到）
- 忙着跟 ChatGPT 争 C 端，给了 Anthropic 黄金窗口

</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">

**优势**
- 算力最足、现金流最足（worst case TPU 变另一个英伟达）
- 资源布局无短板，战略跟随即可
- 已到第三代职业经理人，体系化运转
- 有操作系统、Google Workspace
- 长期看最稳，掉队可能性最低

</div>
