<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed mb-3">
<strong>Copilot 订阅新增 5M</strong>，总付费席位达 20M——但相比 Microsoft 365 的 4.5 亿总席位，渗透率仍不足 5%
</div>

</div>

<div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed mb-3">
<strong>OpenAI 关系重构</strong>：Azure 失去独家销售权，但 Microsoft 保留了 OpenAI 的股权收益。Satya Nadella 重新谈判合同——用 2032 年固定期限替代了模糊的 AGI 条款

</div>

</div>

<div class="mt-3 p-3 bg-purple-50 border rounded text-sm leading-relaxed">
<strong>作者概括：</strong>Microsoft 面临两股力量的博弈——Azure 销售团队失去了"OpenAI 独家"这张王牌，但 OpenAI 在更多平台上的增长最终会反哺 Microsoft 的股权投资回报。

# 蒸馏指控：AI 行业的公开秘密

<div class="mt-4 p-4 bg-red-50 border-2 border-red-200 rounded text-base leading-relaxed">
<div class="text-sm opacity-60 mb-1">庭审中最具爆炸性的时刻</div>

</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
<strong>OpenAI 的论点：</strong>Elon 一边说这是毁灭世界的技术，一边却在"偷"来用于自己的营利公司——典型的言行不一

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 text-sm leading-relaxed">
<strong>Elon 的回应：</strong>"每个人都在某种程度上这样做"——这是行业的公开秘密。这个说法虽然"可能有一定道理"，但"不能让它变得合法"
</div>

</div>

<div class="mt-3 p-3 bg-gray-50 border rounded text-sm leading-relaxed">
<strong>作者概括：</strong>这个指控的戏剧性不在于它是否成立（蒸馏在 AI 行业确实普遍），而在于它把 AI 行业最不愿公开讨论的竞争手段搬到了法庭上。

# 庭审作为剧场：为什么陪审团审判如此危险

<div class="text-sm leading-relaxed mt-2 space-y-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500">
<strong>法官的权力：</strong>法官可以推翻陪审团裁决，实际负责决定赔偿金额——但法官通常不愿推翻陪审团
</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">
<strong>AI 安全证人：</strong>Musk 方传唤 Stuart Russell 作证——试图论证 AI 的危险性，但法官禁止过度渲染"末日论"

</div>

::right::

<div class="flex justify-center items-center h-full pl-4">

<div class="text-sm leading-relaxed space-y-3">

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500">

# 国防科技：Andoril 与百年军事工业史

<div class="mt-4 p-3 bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">
Kyle Harrison（Contrary 合伙人）耗时两年写作《The Andoril Thesis》，追溯 Andoril 的战略定位如何被近一个世纪的军事工业历史所塑造。
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed mb-3">
<strong>"创始人 + 军事异见者"配对：</strong>从 Eisenhower + Bernard Treiber（ICBM 发明者）到现代——国防领域的每一次重大突破都依赖这种双人驱动
</div>

<div class="p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
<strong>关键转折：</strong>DOD 一度占全球研发投入的 36%。一旦这种主导地位消退，就需要"创始人模式"来填补创新真空
</div>

</div>

# 非工程师的 Codex 三件事

<div class="mt-4 text-sm opacity-60 mb-3">Alexander 给非技术用户的入门路径：</div>

<div class="space-y-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">
<strong>第一步 · 连接你的工作工具：</strong>把 Codex 连到 Slack/Teams/邮件——问它"我有没有需要紧急回复的消息？"、"帮我总结这个长线程，我应该做什么？"、"搜索公司信息，告诉我 XYZ 是什么"
</div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm leading-relaxed">
<strong>第二步 · 委托一个需要时间的事务：</strong>"帮我在频道里发帖问谁想参加这个会议，然后自动把回复的人加到会议邀请里"——代理会监控回复、回复确认、发送会议链接
</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500 text-sm leading-relaxed">
<strong>第三步 · 让它每天早上自己运行：</strong>一旦你验证了某个流程有效——告诉 Codex "每天早上 7 点做这个"。不需要"请求功能"，直接请求代理
</div>

</div>

<div class="mt-4 p-3 bg-yellow-50 border rounded text-sm leading-relaxed">
<strong>Alexander 的总结：</strong>就像带一个新人——先给简单任务建立信任，再给更难的任务，最后说"自己想办法"。Codex 的路径完全一样：<strong>Easy → Hard → Automated</strong>
</div>
