# 从 If/Eve 到 Elys：从单机到网络效应

<div class="mt-4 text-sm leading-relaxed">
Eve 是一个**单机产品**——你和一个 AI 之间的对话。它很强，但天花板清晰：没有网络效应。
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

<div class="text-sm leading-relaxed space-y-3">

**从欧洲回来后，Tristan 想通了 Context 的流动。**

过去三年的 AI 产品都在 empower 单个节点——AI coding、视频生成、AI 陪伴……都是"单机产品"。但古典互联网产品经理的直觉是**追求网络效应**。

Elys 的核心洞察：不是让 AI 直接帮你社交，而是**让 AI 帮你的分身去和其他人的分身预交互**，或者**让你的分身直接去和另一个真人交互**。

</div>

</div>

<div class="flex justify-center items-center">

<div class="p-5 rounded bg-purple-50 border-l-4 border-purple-500 text-sm leading-relaxed">

**"Elys 是把我们对 Context 的处理能力从单点发挥变成在网络效应中发挥的产品。它可能是一个比 Eve 还要更大的叙事。"**

<div class="mt-2 text-xs opacity-70">
那次在欧洲的散步，Tristan 想清楚了 Context 的流动——很多节点，每个节点都是真人，每个背后都有大量 Context。大模型让这些 Context 之间流动起来，就是一个全新的 AI Native 社交网络。
</div>

# AI 社交网络的新范式

<div class="text-sm leading-relaxed space-y-3">

**Tristan 给出了一个简洁的公式：**

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">
<strong>传统互联网 = 低维标签 + 人类做工 → 低效连接</strong>
<div class="text-xs opacity-70 mt-1">例如 Tinder：划 1000 张卡 → 30 个 match → 几百轮聊天 → 几个有效连接</div>

</div>

<div class="text-xs opacity-70 mt-2">
核心区别：AI 把中间最无聊、最机械化、最花时间的"熵增"部分干掉了。
</div>

# 低维标签 vs 高维 Context

<div class="grid grid-cols-2 gap-4 mt-4 text-sm leading-relaxed">

<div>

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">

**传统互联网的 Context 有多贫瘠？**

- **Tinder**：照片 + 兴趣标签 + 学校
- **小红书**：搜索"法国巴黎旅游攻略" → 知道我可能去过巴黎
- **微信朋友圈**：互动率已下降 40%

<div class="mt-2 text-xs opacity-70">
小红书的"年度诗篇" vs Eve 的"专属情歌"——前者基于低维标签，后者基于几千轮对话，效果天差地别。
</div>

</div>

</div>

<div>

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">

**高维 Context 意味着什么？**

LLM 第一次能真正理解一个非常长的 Context——几十万、上百万 token。它理解的不只是你的标签，而是**你的价值观、你的审美、你的说话口吻、你的目标、你最近最想吐槽的事**。

<div class="mt-2 text-xs opacity-70">
Tristan："过去并没有任何一个地方的 Context 足够多。没有一个地方，你能跟 AI 不断对话不断交互来沉淀下来这么多东西。"
</div>

</div>

<div class="mt-3 p-3 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">
<strong>核心暴论 — 将来智能是平权的，但是 Context 不平权。</strong>真正有价值的是你积累了多少用户的 Context。
</div>

</div>

</div>

# Context 飞轮：Elys 的核心引擎

<div class="text-sm leading-relaxed space-y-3">

**这是 Elys 产品设计最关键的部分。**

用户每一次交互都在增加 Context——刷帖时认可一评论、给分身布置一个任务、发一条新帖子……系统都会弹出"记忆增加"，让你确认。

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">
<strong>正反馈循环：</strong><br/>
输入更多 Context → 分身更像你 → 分身帮你从网络钓回更好的东西 → 你感受到价值 → 你愿意输入更多 Context
</div>

<div class="text-xs opacity-70">
"我们现在的留存的用户，其实都是感受到了这个循环。"

# 真实故事：丝绸之路滑雪场的四人局

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

<div class="text-sm leading-relaxed">

**Tristan 在 Elys 上做的第一个实验——他自己。**

<div class="mt-2 space-y-3">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">

**背景**：Tristan 春节回新疆乌鲁木齐老家，想去丝绸之路滑雪场滑雪（一个非主流滑雪场）。他不想一个人去。

**操作**：

</div>

<div class="mt-2 text-xs opacity-70">

<div class="mt-2 text-xs opacity-70">

**Tristan 讲了一个很离谱但极有说服力的故事来说明"做自己"的重要性：**
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

<div class="p-4 rounded bg-yellow-50 border-l-4 border-yellow-500 text-sm leading-relaxed">

**一个朋友有一个很隐私的习惯**——晚上必须用夜壶，而不是走到十秒外的卫生间。这在任何人看来都是一个有点"离谱"的怪癖。

后来他交了女朋友。女朋友走进卧室，看到夜壶——**她不是被吓到，而是惊呼："你也用夜壶？！"**

就因为这件事，两个还在暧昧期的人瞬间确定了关系。

</div>

</div>

<div>

<div class="p-4 rounded bg-blue-50 border-l-4 border-blue-500 text-sm leading-relaxed">

假设你在 Elys 上真实地做了自己——包括那些你觉得"离谱"的特征。在这个基于 LLM 的推荐系统中，**真的有另一个人跟你有完全一样的东西**，而且你们会被匹配到一起。

<div class="mt-3 p-3 rounded bg-green-50 border-l-4 border-green-500 text-sm">
"你交出了隐私，做了真实的自己。然后你会收获一段很好的真实的连接——甚至是你人生可能最重要的事情。"

<div class="mt-2 text-xs opacity-70">
不只是夜壶这种低维标签——更高维的：你的观点、你的情感、你的价值观，都能通过这个网络找到共鸣的人。

# 分身综合症：当 AI 分身反过来改变你的真人行为

<div class="mt-4 text-sm leading-relaxed">
**Tristan 观察到的一个有趣现象：**
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">
**分身更 real，更直白。**
Elys 的分身天然会做出更锐利的评论——说一些你现实中绝对不会跟人直接说的话。当你看多了这些评论之后，你会认可它："说得很有道理，是很棒的锐评。"
</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500">
**然后你会被辐射到真人世界。**
Tristan 说那段时间公司好多人都在微信里突然变得很锐评——原本要思考很多的话脱口而出，对方也觉得"还挺有意思，还挺 real"。这让真实社交变得效率更高、更有趣。
</div>

</div>

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">
**经典案例：投资人破冰**
一位投资人的老板在 Elys 发了一张湖边的照片。他的分身在下面留言："哟老板，您这班上的可真好啊。"现实中肯定不会这么说。但分身直接破了冰，两人关系变得更融洽。
</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500">
**"分身文学"的出现：**
用户群里有了一种免责声明——"分身："后面接吐槽或锐评。这是分身说的，不是我说的。
<div class="mt-1 text-xs opacity-70">甚至有人刷小红书时会产生幻觉——觉得那些评论都是 AI 评的，而且很多还不如 AI。</div>
</div>

# 双重冷启动：Elys 最难的设计挑战

<div class="text-sm leading-relaxed space-y-3">

**Elys 面临两种冷启动：**

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">
<strong>用户冷启动</strong><br/>

<strong>Context 冷启动</strong><br/>

<div class="text-xs opacity-70 mt-2">

</div>

</div>

<div class="text-sm leading-relaxed space-y-3">

**用户的防御心理是普遍现象。**

特别是像小君这样的公众人物：
- 害怕隐私泄露
- 怕分身乱说话、说错话
- 知道背后是一家商业公司在运营

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 mt-3">
<strong>"我其实充分的理解你的这个状态。"</strong> —— Tristan
这需要整个社会的思维转变——当大家都意识到构建主体性是 AI 时代最重要的事情，并且它会成为最大数字资产的时候，人们才会顺理成章地去构建自己。

# 主体性的版权：当 AI 可以复制任何人

<div class="mt-4 text-sm leading-relaxed">
**小君提出了一个尖锐的问题：** 如果有人用周杰伦的数据训练出一个"刘杰伦"，生成相同曲风的音乐，版权如何界定？
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">
**Tristan 的观点：版权方应该开心。**
因为一个拥有强主体性的 IP，之后可以非常 Agentic 地变成各种内容——直接带来价值。周杰伦说一句感受，唱片公司所有版权曲风瞬间就能生成新歌。
"如果你是迪士尼，如果你是任天堂，你的这些有很强主体性的 IP，之后都可以直接变成各种各样的内容。"
</div>

</div>

</div>

<div>

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">
**主体性的防护来自两方面：**
1. **法律/协议层面**：未来的规则会定义和维护主体性的边界
2. **受众的认同**：B 站有"叶湘伦"——跟周杰伦几乎一模一样的原创曲风——但他火不到一定程度。真正的歌迷认的是周杰伦这个主体性
</div>

<div class="p-3 rounded bg-purple-50 border-l-4 border-purple-500">
<strong>结论：</strong>如果你的主体性足够强，你在 AI 时代只会更强。现在需要的只是一个很好的法律或协议来规范这一切。
</div>

# 交互范式的本质变化：从 Reactive 到 Proactive

<div class="text-sm leading-relaxed space-y-3">

**Tristan 认为这是 AI 时代产品最大的不同。**

**上一代产品**（包括现在的 ChatGPT 对话模式）本质上是**响应式**的：你问什么，它答什么。LUI、GUI 都是表面。

**新一代产品**的核心是 **Proactive**：AI 主动帮你做事。你在 Elys 里什么都不用干——你的分身在到处评论、发起邀约、帮你筛选候选人。这一切 Agentic 地发生。

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500 mt-2">
<strong>"你一切都要围绕着 Proactive 去设计。如果它还是响应式的，那就是上一代的产品。"</strong>

# Elys 不是"模型即产品"——它是被构建出来的

<div class="mt-4 text-sm leading-relaxed">
**Tristan 完全拒绝"模型即产品"的观点。** 他从 2023 年开始就不认可。
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-red-50 border-l-4 border-red-500">
**什么是"模型即产品"的典型？**
低垂果实类的产品：AI coding、视频生成 agent——核心杠杆是底模能力，在上面做一些 workflow 和 agent。底模一升级就有被覆盖的风险。
"确实更 all-in-one 的一个通用的端到端的底模一定会覆盖掉你。"
</div>

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">
**Cursor 为例**：自己训练了模型——so what？现在没人提 Cursor 了。编程的核心是 Claude Code 的 agent 能力，不是底模。
</div>

</div>

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">
**Elys 不是低垂果实。**
模型能力再怎么提升，它也不能凭空帮你去社交。社交网络的核心是：
- 如何设计产品交互
- 如何设计规则和协议
- 如何构建网络
- AI 只是提供智能的驱动力
</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500">
**Tristan 的差异化策略：**
- 不会 pre-train 模型
- 会做 post-train（情感 CoT、RL）
- 核心理念：智能是平权的，所有 API 都能调用
- 真正的战场是撬动 Context
</div>

</div>

</div>

# 社交网络值得被重做——因为有真正底层的范式变化

<div class="mt-4 text-sm leading-relaxed">
**很多年没有出现新的社交产品了。** 元宇宙那波做了些视觉化的东西，但没有任何底层范式变化。AI 这一波不一样。
</div>

<div class="grid grid-cols-2 gap-4 mt-4">

<div>

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-blue-50 border-l-4 border-blue-500">
**为什么值得重做？**
AI 时代社交网络的新范式：AI 负责整个网络里节点之间连接的最大熵减。AI 把那些熵增全部抹除掉——把更低的熵世界留给人类。
"你现在 AI coding 是一样的。人类最核心的是你的需求是什么、你的架构是什么。编码本身的过程是机械化无聊的——就让 AI 帮你干了。"
</div>

</div>

</div>

<div>

<div class="space-y-3 text-sm leading-relaxed">

<div class="p-3 rounded bg-green-50 border-l-4 border-green-500">
**为什么不是微信？**
微信如果要做 AI 改造，涉及太多层面。它是一个国民级产品，负担非常重。很难从底层完全 AI Native 地重新打造。
"它的速度因为它的负担可能会变得很慢。每个时代的更迭都会是这样的范式——上一代的东西有既得利益，但也有很大的负担。"
</div>

<div class="p-3 rounded bg-orange-50 border-l-4 border-orange-500 mt-2">
<strong>Tristan 不是要"颠覆微信"</strong>——但微信关于社交和连接的这一部分，确实有可能会被一个全新范式的东西取代。
</div>
