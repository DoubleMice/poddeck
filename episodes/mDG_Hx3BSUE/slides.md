# Alchian-Allen 效应：为什么最好的模型吃掉所有收入

<div class="mt-2 text-sm opacity-60">一个经济学原理解释了为什么 AI 行业的收入集中在最强模型上。</div>

<div class="grid grid-cols-2 gap-6 mt-4">

<div>

### 经济学类比

Dwarkesh 引入了 **Alchian-Allen 效应**：当所有商品的固定成本上升时，消费者倾向于选择更高质量的商品。

<div class="p-3 bg-blue-50 border-l-4 border-blue-500 rounded mt-3 text-sm">

**苹果的例子**：
- 好苹果 $2 vs 差苹果 $1 → 比价 2:1
- 加 $1 关税 → $3 vs $2 → 比价 1.5:1
- 好苹果"相对"更便宜了

</div>

<div class="p-3 bg-green-50 border-l-4 border-green-500 rounded mt-3 text-sm">

**应用到 AI**：
GPU 成本上升是固定成本 → Opus 和 Sonnet 的价差缩小 → 用户更愿意多花一点用最好的模型

</div>

</div>

<div>

<div class="p-4 bg-orange-50 border-l-4 border-orange-500 rounded text-sm italic">

Dylan 补充的例子：一台 Hopper 能产 100 万 Opus token 或 200 万 Sonnet token。GPU 从 $2 涨到 $3 后，Opus 和 Sonnet 的价差缩小了，用户自然倾向选 Opus。

</div>

<div class="mt-4 p-3 bg-yellow-50 border-l-4 border-yellow-500 rounded text-sm">

**Dylan 的数据验证**：当前所有的营收和流量都集中在最好的模型上。在一个算力受限的世界里，用户宁愿多付一点用最强模型，也不愿意用便宜但弱的。

</div>

<div class="mt-4 p-3 bg-red-50 border-l-4 border-red-500 rounded text-sm">

**推论**：Anthropic 面临的不是需求不足——而是必须"destroy demand"（销毁需求），因为产能跟不上。当前的增速下，他们无法不限制需求。

# EUV 数学：3.5 台工具 = 1 GW

<div class="text-sm opacity-60 mb-3">从晶圆到数据中心，一个简单的算术就能算出 AI 扩张的天花板。</div>

<div class="text-sm space-y-2">

**制造 1 GW Rubin 芯片需要**：
- ~55,000 片 3nm 晶圆（每片 20 次 EUV）

# 为什么不用普通 DRAM 替代 HBM？

<div class="mt-2 text-sm opacity-60">Dwarkesh 问了一个好问题：Agent 不需要秒回，那能不能用便宜的 DDR 做慢速推理？</div>

<div class="grid grid-cols-2 gap-6 mt-4">

<div>

### 带宽才是真正的瓶颈

<div class="p-3 bg-blue-50 border-l-4 border-blue-500 rounded text-sm">

**HBM4**（Rubin 使用）：
- 2048 bits 宽，13mm shoreline
- ~10 GT/s 传输速率
- = **2.5 TB/s** per stack

</div>

<div class="p-3 bg-orange-50 border-l-4 border-orange-500 rounded text-sm mt-2">

**DDR5**（同等 shoreline）：
- 64-128 bits 宽
- ~8 GT/s
- = **64-128 GB/s**

</div>

<div class="mt-2 text-sm font-bold text-red-600">
→ 同样的芯片边缘面积，HBM 带宽是 DDR 的 ~20 倍
</div>

</div>

<div>

### 真正重要的指标

<div class="p-3 bg-purple-50 border-l-4 border-purple-500 rounded text-sm">

Dylan 纠正了一个常见误区：你真正关心的指标是 **bandwidth per wafer**（每片晶圆的带宽），而不是 bits per wafer（每片晶圆的存储位）。

因为限制 FLOPS 利用率的是把下一个矩阵搬进搬出的速度。

</div>

<div class="mt-3 p-3 bg-yellow-50 border-l-4 border-yellow-500 rounded text-sm">

**为什么慢模式行不通**：即使 Anthropic 可以出一个"慢速 Opus"模式（速度 2x 慢、价格 4-5x 便宜），实际上没人想用——最高价值的任务都有时间敏感性。Agent 任务从 1 小时变成 1 天，没人愿意等。

</div>

</div>

# 为什么小模型反而跑得更快

<div class="mt-2 text-sm opacity-60">Dylan 解释了为什么 AI 实验室倾向于训练更小的模型 + 更多 RL，而不是暴力堆参数。</div>

<div class="grid grid-cols-2 gap-6 mt-4">

<div>

### 算力分配的三个桶

<div class="space-y-2 text-sm">
<div class="p-3 bg-blue-50 border-l-4 border-blue-500 rounded">

**推理（Inference）** = 收入
<div class="text-xs opacity-70 mt-1">产出 token → 赚钱 → 融更多钱</div>

</div>
<div class="p-3 bg-green-50 border-l-4 border-green-500 rounded">

**研发（Research）** = 效率提升
<div class="text-xs opacity-70 mt-1">研究员尝试新想法 → 推动 Pareto 前沿 → 模型成本年降 10x</div>

</div>
<div class="p-3 bg-orange-50 border-l-4 border-orange-500 rounded">

**开发（Development）** = 预训练 + RL
<div class="text-xs opacity-70 mt-1">实际训练下一个模型</div>

</div>
</div>

</div>

<div>

### 小模型的复合优势

<div class="p-4 bg-purple-50 border-l-4 border-purple-500 rounded text-sm">

假设 5T 参数模型比 1T 模型的采样效率高 2x：

- 5T 模型每次 RL rollout 需要 5x 计算
- 所以等量 RL 用时 2.5x
- 1T 模型虽然采样效率低，但能做 2x 的 rollout
- **1T 模型更快完成** → 更早部署到研发辅助 → 研究员更快产出下一代

</div>

<div class="mt-3 p-3 bg-yellow-50 border-l-4 border-yellow-500 rounded text-sm">

**复合效应**：小模型 → 更快 RL → 更早部署 → 加速研究 → 更好的算法 → 下一个模型更高效...这个反馈循环才是所有实验室追求的"最快起飞"。

</div>

<div class="mt-2 text-xs opacity-60">

例外是 Google——他们有统一的 TPU fleet 和大型 scale-up 域，所以能让大参数模型（如 Gemini Pro）的 RL 速度也足够快。

</div>

</div>
